Frequently Asked Questions

What is Cypher?

Cypher is an open graph query language that allows for expressive and efficient querying of graph data. Cypher is intuitive, powerful and easy to learn. It lets you write graph queries by describing patterns in your data. Because graphs describe your domain, Cypher lets you focus on your domain instead of getting lost in the mechanics of data access.

Cypher is designed to be a human readable query language, suitable for both developers and operations professionals. The guiding principle behind Cypher is to make simple things easy, and complex things possible. Cypher combines English prose and neat iconography which helps to make queries more self-explanatory.

Cypher is declarative, which means that it lets users express what data to retrieve, letting the engine underneath Cypher take care of seamlessly retrieving it. This is in contrast to imperative or scripting languages, including early-generation graph languages, which require query writes to have a deep technical understanding of physical implementation details.

Cypher inspired by a number of different approaches and builds upon established practices for expressive querying. Most of the keywords like WHERE and ORDER BY are inspired by SQL. Pattern matching borrows expression approaches from SPARQL. Some of the collection semantics have been borrowed from languages such as Haskell and Python.

What are the benefits of openCypher?

The openCypher project will make it possible for any developer or technology provider to use Cypher and incorporate and/or enable access to graph processing capabilities within that product or application. openCypher will also help tooling providers gain efficiencies by gaining multi-platform.

openCypher delivers Cypher as an open graph query language, which can be used by any product or application as a query language for graph processing capabilities within that product or application. As an easy-to-learn and yet an extremely powerful graph query language, Cypher is currently the industry’s most successful and widely adopted graph query language and with openCypher, the language promises to be instrumental in the growth of graph processing and analysis in the same way that SQL has been instrumental in accelerating the adoption of RDBMS.

What will be delivered as part of openCypher?

The project aims to deliver four types of artifacts:

  • Cypher Reference Documentation - comprehensive user documentation describing use of the Cypher query language, with examples & tutorials
  • Technology certification kit (TCK) - The TCK consists of a number of tests that a software supplier would run in order to self certify support for a given version of Cypher
  • Reference implementation - Distributed under the Apache 2.0 license, the reference implementation is a fully functional implementation of key parts of the stack that one would need to build to support Cypher inside of one’s data platform or tool. The first planned deliverable is a parser that will take a Cypher statement and parse it into an AST (abstract syntax tree) representation. The reference implementation complements the documentation and tests by providing working implementations of Cypher, that are permissively licensed, and can be used as examples or as a foundation for one’s own implementation.
  • Cypher language specification - Licensed under the creative commons license <>, the Cypher language specification is a technical expression the language syntax to enable parsers to auto-generate the query syntax. A full semantic specification is also planned as a part openCypher.

Who is behind openCypher?

openCypher includes support from industry heavyweights such as Oracle, SAP, Microsoft and IBM. Other early supporters include ThoughtWorks, Tableau, and DataBricks, the company behind Apache Spark.
